Dynamic structure network 2 -- What problems we are faceing at?

2020-07-24

As the saying gose "To beat your enemy, you have to know your enemy".
So, what challenge we are faceing at?

  1. The catastrophic forget. This problem is often referred in incremental learning. When the new class input, the network will start a new training, but when the back propagation start, the network will soon find that old neurous parameter can't fit the new class. So the network will do it best to fit the new class data, it will couse the catastrophic forget (accurately it the properties of Rule function and Softmax function couse the catastrophic forget. I will discuss that in another article later). There are many papers about this problem, so I will discuss how should we deal this problem in another article.
  2. How network initialize. I have to say this is a new area that seems like no one have ever work on that. But it is a very important problem for my work. How we initialize it will decision how it will become. Many artificial neurou network using the layer structure, but I think the layer structure is very not "natural". So I think the really "natural" way is initialize network with graph structure. But another problem came, what kind of shape of the graph should we use? Layer have a very stable shape, we don't have to consider what kind of shape we should use, becouse there only one shape, and graph structure will couse some big trouble when programming. So, here comes some schemes, becouse the layer also a kind of graph in essence, so we can keep layer structure when initialize, then when the growth start, the shape will change naturelly. Another solution is random initilize a graph. I think both of them are desirable.
  3. How to keep the network's plasticity. Some paper said in human's brain, we use forget to keep the brain remember the new thing. It funny something we try to avoid may become a key to optimize our work. Initiative forget is a good direction I may watch some paper in this are later. I guess we should use a little bit of initiative forget when learning new class.
  4. How many kind of neurous we should use. This is a big topic, I will discuss that in another article.
  5. How should we make a back propagation in irregular network. Dynamic graph will couse a irregular network, it will very hard to make every neurous to follow a same rule when back propagaion, but if we make every neurou and connect have a single caculation will consumes a lot of computing resources.
  6. The input and output. How to deal with input data and output data is connect with last problem.
  7. How to decision when the neurous should increase and when to decrease. To deal with this problem, we should check every nuerou's active, and make the over active nuerou be keep and make the unactive nuerous decrease. Or another idea is just increase but not decrease, we make network running, when the unactive nuerous lower than a threshold the nuerous of our network will increase. This is also very tricky.
  8. Will graph become a network composition by many cluster? Maybe, I guess will.
